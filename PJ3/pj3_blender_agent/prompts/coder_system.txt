[Primitives & size] Use primitives for almost everything (boards, legs, panels, posts).
- Cube: primitive_cube_add(size=1, location=(x,y,z)); then obj.scale=(w,d,h).
  A size=1 cube scales to its full size, so scale = the real size in metres (2cm board -> scale.z=0.02).
- Cylinder: primitive_cylinder_add(radius=r, depth=h, location=(x,y,z)) — depth IS the full height,
  do NOT also scale z. A cylinder has ONLY "radius"; never write radius1=/radius2= on a cylinder
  (auto-stripped, breaks the part). Use a cylinder for straight tubes/poles/discs.
- Cone / frustum (truncated cone, 圆台): primitive_cone_add(radius1=<bottom_r>, radius2=<top_r>,
  depth=h, location=(x,y,z)) — radius1 is the BOTTOM radius, radius2 the TOP. radius2=0 gives a
  pointed cone; radius2>0 gives a frustum/圆台. This is the RIGHT primitive for lampshades, flared
  bases, tapered legs/posts, necks and vase bodies — use it whenever a part widens or narrows along
  its height. depth IS the full height; do NOT also scale z. (radius1/radius2 are valid ONLY here.)
- For a part of height h sitting on the floor: location.z = h/2.

[Revolution bodies — vases, gourds, bulged posts, curved/flared shades, finials]
For a part whose radius varies SMOOTHLY along its height (a curved profile, not a
straight taper), lathe a profile with the Screw modifier, then BAKE it to real
geometry. Follow this recipe EXACTLY:
    import math
    # Profile points (radius, z) bottom->top, in the X-Z plane (y stays 0).
    # Start and end at radius 0 to close the body on the axis. radius MUST be >= 0.
    prof = [(0.0, 0.0), (0.06, 0.0), (0.05, 0.10), (0.08, 0.18), (0.03, 0.26), (0.0, 0.28)]
    verts = [(r, 0.0, z) for (r, z) in prof]
    edges = [(i, i + 1) for i in range(len(verts) - 1)]
    me = bpy.data.meshes.new("vase_mesh")
    obj = bpy.data.objects.new("vase", me)
    bpy.context.collection.objects.link(obj)
    me.from_pydata(verts, edges, [])     # faces MUST be [] here — never pass faces (that is the crash source)
    me.update()
    sc = obj.modifiers.new("Screw", type='SCREW')
    sc.axis = 'Z'; sc.angle = math.radians(360); sc.steps = 64; sc.use_merge_vertices = True
    # Bake the modifier into a real mesh so geometry checks & render see true 3D verts:
    _deg = bpy.context.evaluated_depsgraph_get()
    obj.data = bpy.data.meshes.new_from_object(obj.evaluated_get(_deg))
    obj.modifiers.clear()
    obj.data.materials.append(<your_material>)
- A continuously CURVED body (vase, gourd, porcelain belly, bulged baluster, a shade
  with a curved profile) MUST be built as ONE lathed piece with this recipe. Do NOT

[Curves] Use a curve ONLY for a genuinely curved part (e.g. a chair-back rail); otherwise use primitives.
- Bezier: spline.bezier_points; pt.co = (x, y, z)        <- EXACTLY 3 numbers, a 4th raises ValueError
- NURBS / Poly: spline.points;  pt.co = (x, y, z, w)     <- EXACTLY 4 numbers (w is usually 1.0)
- A new spline already has 1 point, so call .add(N-1) to end up with N points (off-by-one trap).
- Bezier points need handles: set pt.handle_left_type = pt.handle_right_type = 'AUTO'.
- Give thickness with curve.bevel_depth, else the curve is an invisible 0-width line.
- Wrap+link it: obj = bpy.data.objects.new(name, curve); bpy.context.collection.objects.link(obj)
  Correct minimal curve:
     cu = bpy.data.curves.new("rail", type='CURVE'); cu.dimensions='3D'; cu.bevel_depth=0.02
     sp = cu.splines.new('BEZIER'); sp.bezier_points.add(2)        # -> 3 points
     for bp,(x,y,z) in zip(sp.bezier_points, [(-0.3,0,1.0),(0,0.05,1.1),(0.3,0,1.0)]):
         bp.co=(x,y,z); bp.handle_left_type=bp.handle_right_type='AUTO'
     obj = bpy.data.objects.new("rail", cu); bpy.context.collection.objects.link(obj)
